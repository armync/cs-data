De ce este POO o paradigm bun?

Succesul acestei paradigme se poate datora i faptului c multe dintre cele mai folosite limbaje se ntampl s fie limbaje orientate pe obiect, cum ar fi:

Android iOS Web
Java Swift JavaScript
C++ Objective-C Python
Kotlin PHP
Ruby

Spre deosebire de programarea procedural, care folosete o list de instruciuni pentru a spune computerului pas cu pas ce s fac, programarea orientat-obiect se folosete de componente din program care tiu s desfoare anumite aciuni i s interacioneze cu celelalte.

Astfel, POO ofer modularitate i ordine programului, reuind s modeleze situaii din viaa real - fiind mai avansat dect programarea procedural, care reflect un mod simplu, direct de a rezolva problema.

De ce s folosim POO?

  • Scrierea de cod complex devine clar i cu minim de erori (readability)
  • mprirea codului ntre membrii echipei i urmrirea progresului (transparency)
  • Cod uor de extins i reparat (extensibility)
  • Devine clar ce cod este testat automat i ce cod nu este testat (testability)
  • Refolosim soluii la probleme comune i uneori chiar cod (reusability)
  • Uurina n crearea - uneori automat - a documentaiei (documentation)

Clase

Clasele reprezint tipuri de date definite de utilizator sau deja existente n sistem (din class library - set de biblioteci dinamice oferite pentru a asigura portabilitatea, eliminnd dependena de sistemul pe care ruleaz programul). O clas poate conine:

  • membri - variabile membru (cmpuri) i proprieti, care definesc starea obiectului
  • metode - functii membru, ce reprezint operaii asupra strii.

Prin instanierea unei clase se nelege crearea unui obiect care corespunde unui ablon definit. n cazul general, acest lucru se realizeaz prin intermediul cuvntului cheie new.

Procesul de iniializare implic: declarare, instaniere i atribuire. Un exemplu de iniializare este urmtorul:

Un alt exemplu de clas predefinita este clasa String. Ea se poate instania astfel (nu este necesar utilizarea new):

Aceasta este varianta preferat pentru instanierea string-urilor. De remarcat c i varianta urmtoare este corect, dar ineficient, din motive ce vor fi explicate ulterior.

Cmpuri (membri)

Un cmp este un obiect avnd tipul unei clase sau o variabil de tip primitiv. Dac este un obiect atunci trebuie iniializat nainte de a fi folosit (folosind cuvntul cheie new).

     float f;

     String s; 
 } 
 

Declarm un obiect de tip DataOnly i l iniializm:

Observm c pentru a utiliza un cmp/funcie membru dintr-o funcie care nu aparine clasei respective, folosim sintaxa:

Proprieti

O proprietate este un cmp (membru) cruia i se ataeaz dou metode ce i pot expune sau modifica starea. Aceste doua metode se numesc getter si setter.

     String myString;

         return myString;
     }

         myString = s;
     }

Declarm un obiect de tip PropertiesExample i i iniializm membrul myString astfel:

Metode (funcii membru)

Putem modifica programul anterior astfel:

 s2 = "My second string";

Va fi afiat lungimea n caractere a irului respectiv. Se observ c pentru a aplica o funcie a unui obiect, se folosete sintaxa:

Funciile membru se declar asemntor cu funciile din C.

Specificatori de acces

n limbajul Java (i n majoritatea limbajelor de programare de tipul OOP), orice clas, atribut sau metod posed un specificator de acces, al crui rol este de a restriciona accesul la entitatea respectiv, din perspectiva altor clase. Exist specificatorii:

Tip primitiv Definitie
private limiteaz accesul doar n cadrul clasei curente
default accesul este permis doar n cadrul pachetului (package private)
protected accesul este permis doar n cadrul pachetului si n clasele ce motenesc clasa curent
public permite acces complet

Atenie, nu confundai specificatorul default (lipsa unui specificator explicit) cu protected

ncapsularea se refer la acumularea atributelor i metodelor caracteristice unei anumite categorii de obiecte ntr-o clas. Pe de alt parte, acest concept denot i ascunderea informaiei de stare intern a unui obiect, reprezentat de atributele acestuia, alturi de valorile aferente, i asigurarea modificrii lor doar prin intermediul metodelor.

Utilizarea specificatorilor contribuie la realizarea ncapsulrii.

ncapsularea conduce la izolarea modului de implementare a unei clase (atributele acesteia i cum sunt manipulate) de utilizarea acesteia. Utilizatorii unei clase pot conta pe funcionalitatea expus de aceasta, indiferent de implementarea ei intern (chiar i dac se poate modifica n timp).

Exemplu de implementare

Clasa VeterinaryReport este o versiune micorat a clasei care permite unui veterinar s in evidena animalelor tratate, pe categorii (cini/pisici):

     int dogs; 
     int cats;

         return dogs + cats;
     } 
 
     public void displayStatistics() { 
         System.out.println("Total number of animals is " + getAnimalsCount());
     }

Clasa VeterinaryTest ne permite s testm funcionalitatea oferit de clasa anterioar.

         VeterinaryReport vr = new VeterinaryReport();

Observaii:

  • Dac nu iniializm valorile cmpurilor explicit, maina virtual va seta toate referinele (vom discuta mai mult despre ele n laboratorul urmtor) la null i tipurile primitive la 0 (pentru tipul boolean la false).
  • n Java fiierul trebuie s aib numele clasei (publice) care e coninut n el. Cel mai simplu i mai facil din punctul de vedere al organizrii codului este de a avea fiecare clas n propriul fiier. n cazul nostru, VeterinaryReport.java i VeterinaryTest.java.

Obiectele au fiecare propriul spaiu de date: fiecare cmp are o valoare separat pentru fiecare obiect creat. Codul urmtor arat aceast situaie:

     public static void main(String[] args) { 
         VeterinaryReport vr = new VeterinaryReport(); 
         VeterinaryReport vr2 = new VeterinaryReport();
 
         vr.cats = 99; 
         vr.dogs = 199; 
         vr2.dogs = 2;

         vr2.displayStatistics();

         System.out.println("The second class method says there are " + vr2.getAnimalsCount() + " animals");

Se observ c:

  • System.out.println() este metoda utilizat pentru a afia la consola standard de ieire
  • Operatorul '+' este utilizat pentru a concatena dou iruri
  • Avnd n vedere cele dou observaii anterioare, observm c noi am afiat cu succes irul The first/second class method says there are + vr.getAnimalsCount() + animals, dei metoda getAnimalsCount() ntoarce un ntreg. Acest lucru este posibil deoarece se apeleaz implicit o metod care convertete numrul ntors de metod n ir de caractere. Apelul acestei metode implicite atunci cnd chemm System.out.println se ntmpl pentru orice obiect i primitiv, nu doar pentru ntregi.

Spre deosebire de C, n Java instruciunile trebuie scrise n cadrul metodelor (funciilor), iar metodele trebuie implementate n clase.

Semntura main-ului

n Java, funcia main are urmtoarea semntur: public static void main(String[] args), care este echivalent cu int main (int argc, char* args[]) din C/C++. n Java, funcia main nu returneaz nimic (este void) i este o metod static (o s vedem despre static n urmtoarele laboratoare). Este important s pstrm aceast semntur pentru ca s fie recunoscut drept entry point-ul programului.

Instanierea obiectelor i popularea cmpurilor unui obiect

O greeal frecvent este atunci cnd se acceseaz cmpul unui obiect, ns acel obiect nu este instaniat, fapt ce duce la apariia unei erori (excepia NullPointerException, care este echivalent cu segmentation fault din C). Cnd vrem s folosim un obiect, mai nti il instaniem (adic l crem cu ajutorul keyword-ului new - echivalent cu malloc din C), apoi dup instaniere putem s i populm cmpurile cu informaii.

Exemplu de greeal:

// va fi aruncat excepia NullPointerException, avnd n vedere c nu are sens s lucrm cu ceva ce nu exist (adic obiectul st, care nu a fost creat)

Exemplu corect:

st.name = "Costic";

Folosirea array-urilor

n Java, putem folosi array-uri similar ca n C. Pentru a crea un array de 10 numere ntregi, declarm n felul urmtor n Java: int[] arr = new int[10];, arr avnd o dimensiune fix de 10 elemente.

Cnd construim un array de obiecte, trebuie s avem grij ca obiectele din array s fie instaniate, altfel vor putea aprea erori (de regul, excepia NullPointerException).

Exemple de greeli:

// Student arr[3]; // nici aa nu merge declarat un array n Java, aa se poate n C

// aici vom avea NullPointerException, deoarece nu exist obiectul de tip Student din poziia 0, acesta nefiind instaniat

Exemplu corect:

arr[0] = new Student(); // mai nti punem un obiect Student n poziia 0, apoi atribuim valori obiectului din poziia respectiv
arr[0].name = "Andreea";
 
arr[1] = new Student();
arr[1].name = "Maria";
 
arr[2] = new Student();
arr[2].name = "Daniela";

Este important s alocm ct avem nevoie, nu este recomandat s alocm memorie pentru 1000 de elemente i s populm array-ul doar cu 50 de elemente. n plus, pot aprea probleme (excepia NullPointerException, atunci cnd ncercm s accesm un element care nu a fost creat / instaniat).

Pentru a evita probleme de acest tip, putem s folosim colecii, despre care vom vorbi n laboratoarele urmtoare.

Exemplu de greeal:

}

Exemplu corect:

JavaDoc

Ce este?

JavaDoc reprezint o specificaie care explic scopul sau nelesul elementului cruia i este ataat. Acesta se poate ataa fie unei clase, fie unei metode. Codul pe care l scriem nu este complet dac nu are acest tip de documentaie, deoarece, cu toate c urmtoarea persoan poate s i dea seama ce face o bucat de cod, aceasta nu o s aib nicio informaie legat de utilitate sau despre direcia de dezvoltare din viitor. Fr o astfel de documentaie un programator nu poate lua decizii informate despre cum s interacioneze cu codul.

Cum trebuie s fie aceast documentaie?

Atunci cnd scriem documentaia trebuie s inem cont de 3 aspecte. Aceasta trebuie s fie clar, complet i concis.

Structura

Un JavaDoc trebuie s fie uor de citit i astfel recomandm urmtoarea structur:

  • o propoziie care sumarizeaz ce presupune clasa / metoda
  • informaii despre parametrii de intrare ale unei metode i ce ntoarce metoda
  • dac exist referine utile pentru clas / metod

Aceast structur este o sugestie care trebuie adaptat n funcie de unde este folosit.

Coding style for beginners

De ce s respectm un coding style?

Un aspect foarte important n momentul n care trebuie s scriei cod n Java este legat de modul n care scriei, mai exact de organizarea codului n interiorul unor clase cu funcionaliti bine definite. Poate cel mai important motiv al respectrii acestor reguli este faptul c v va fi de ajutor n momentele n care facei debugging sau testing pe o surs. Primii pai pentru a avea un cod ct mai lizibil i uor de urmrit sunt urmtorii:

  • Dai nume intuitive variabilelor i metodelor folosite (de ex: nu folosii nume simpliste, precum 
    int a, b, x, y;

    deoarece la un moment dat vei uita ce rol au acestea i ce reprezint )

  • Metodele folosite trebuie s exprime o funcionalitate bine definit i numai una
  • Implicit, metodele trebuie s nu aib multe linii de cod (vezi Rule of 30) pentru a se putea asigura testarea unei singure funcionaliti mai trziu

Summary

  • Codul Java este scris n interiorul claselor, enum-urilor i interfeelor (lab interfete)
  • Un fiier de tip java poate conine mai multe clase
  • Numele singurei clase publice dintr-un fiier trebuie s fie acelai cu numele fiierului (fr extensia .java)
  • Fiierele sunt grupate n pachete
  • n interiorul unei clase declarm
    • variabile
    • metode
    • alte clase (lab clase interne)
  • Clasele i metodele ar trebui s aib o singur responsabilitate. Evitai metodele lungi i clase cu mai mult de un rol. Nu punei toat logica n metoda main. Metoda main ar trebui s conin doar cteva linii cu instaieri si apeluri de metodele specializate.
  • n declaraia claselor, cmpurilor, metodelor putem aduga specificatori de acces sau diverse keywords pe care le vom studia n urmtoarele laboratoare
    • specificatori de access: public, private, default, protected
  • Java are coding style-ul su i este important respectarea lui, altfel proiectele, mai ales cele cu muli dezoltatori, ar arta haotic.
  • Tipuri de date primitive: int, long, boolean, float, double, byte, char, short. Cmpurile primitive se iniializeaz automat la instanierea unei clase cu valori default: e.g. 0 pentru int.
  • Clasa String ofer multe metode pentru manipularea irurilor de caractere.

Fiind un laborator introductiv i cu multe concepte destul de greu de verificat automat cu LambdaChecker, punctajul de sptmna aceasta este opional/bonus. Cu toate acestea, ncercai s rezolvai toate exerciiile i s punei ct mai multe ntrebri asistenilor, pentru a v nsui ct mai bine cunotinele.

Task 1 (3p)

  1. Creai pachetul lab1, unde adugai codul din seciunea Exemplu de implementare. Rulai codul din IDE.
  2. Folosind linia de comand, compilai i rulai codul din exemplu
  3. Mutai codul ntr-un pachet task1, creat n pachetul lab1. Folosii-v de IDE, de exemplu Refactor Move pentru IntelliJ. Observai ce s-a schimbat n fiecare fiier mutat.

Task 2 (5p)

Creai un pachet task2 (sau alt nume pe care l dorii s l folosii). n el creai clasele:

  • Student cu proprietile: name (String), grade (double)
  • Internship
    • cu proprietile: name (String), minGrade (String), students (array de clase Student - exemplu arrays).
    • cu metoda: chooseCandidateRandomly care returneaza in mod aleatoriu un student din lista de studenti corespunzatoare unui internship (Hint: folositi clasa Random).
    • cu metoda: chooseCandidatesForInterview care va afisa toti candidatii care au obtinut un phone interview la o anumita companie. Fiecare companie are asociat un grade minim dupa care va selecta candidatii pentru un phone interview. Astfel, daca grade-ul unui candidat este mai mare sau egal cu grade-ul minim dorit de o companie pentru angajatii sai, atunci acesta va primi un phone interview. Afisarea candidatilor se va face sub forma: Candidate [nume_student] got a phone interview at [nume_internship]
  • Nu folosii vreun modificator de acces pentru variabile (aka nu punei nimic n faa lor n afar de tip)
  • JobMarket cu metoda main.
    1. Creati studentii cu numele Gigel, Dorel, Marcel si Ionel.
    2. Creati internship-uri cu numele Google, Amazon, Facebook, Microsoft.
    3. Apelati metoda chooseCandidatesForInterview(), pe rand, pentru fiecare companie din cele enumerate mai sus, respectand ordinea lor.
  • Opional, n loc de arrays pentru campul students putei s folosii i obiecte de tip List, e.g. ArrayList (exemplu)

Task 3 (1p)

  1. Creai dou obiecte Student cu aceleai date n ele. Afiai rezultatul folosirii equals() ntre ele. Discutai cu asistentul despre ce observai i pentru a v explica mai multe despre aceast metod.
    • documentaie equals

Metoda equals este folosit n Java pentru a compara dac dou obiecte sunt egale n ceea ce privete informaiile ncapsulate n acestea. Mai precis, se compar referinele celor dou obiecte. Dac acestea indic spre aceeai zon de memorie atunci equals va returna true, altfel va returna false. Vei nva n laboratorul 3 mai multe despre cum se folosete aceast funcie pentru a verifica egalitatea dintre dou obiecte.

Exemplu de folosire: 

if (obj1.equals(obj2)) {
    // do stuff
}

Task 4 (1p)

  1. Adugai modificatorul de acces 'private' tuturor variabilelor claselor Student i Internship (e.g. private String name;)
  2. Rezolvai erorile de compilare adugnd metode getter i setter acestor variabile.
    1. Ce ai fcut acum se numete ncapsulare (encapsulation) i este unul din principiile de baz din programarea orientat pe obiecte. Prin aceast restricionare protejai accesarea i modificarea variabilelor.

Pentru a v eficientiza timpul, folosii IDE-ul (IntelliJ) pentru a generarea aceste metode: Code Generate Getters and Setters
