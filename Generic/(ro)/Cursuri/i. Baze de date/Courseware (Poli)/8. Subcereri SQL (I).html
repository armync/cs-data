Subcereri

Subcererile sunt cereri SQL incluse n clauzele SELECT, FROM, WHERE, HAVING i ORDER BY ale altei cereri numit i cerere principal. Rezultatele ntoarse de subcereri sunt folosite de o alt subcerere sau de cererea principal n situaii cum ar fi:

  • crearea de tabele sau view-uri;
  • inserarea, modificarea i tergerea nregistrrilor din tabele;
  • n furnizarea valorilor pentru condiiile puse n comenzile SELECT, UPDATE, DELETE, INSERT i CREATE TABLE.

Subcererile pot fi imbricate, adic, o subcerere poate avea propriile ei subcereri.

Dac exist o legtur de asociere sau nu ntre subcere i cererea exterioar, subcererile pot fi de 2 tipuri:

  • Subcereri necorelate
  • Subcereri corelate

Din punct de vedere al rolului pe care l au ntr-o comand SQL i a modului de a face construcia comenzii, subcererile pot fi mprite n:

  • Subcereri n clauza WHERE
  • Subcereri n clauza FROM (Subcereri pe tabela temporal)
  • Subcereri n clauza HAVING
  • Subcereri n clauza SELECT
  • Subcereri n clauza ORDER BY

Subcererile pot fi mprite pe mai multe categorii, n funcie de numrul de coloane sau linii pe care le returneaz:

  • Subcereri care ntorc o valoare
  • Subcereri care ntorc o coloan
  • Subcereri care ntorc o linie
  • Subcereri care ntorc mai multe linii

Aceast mprire pe categorii (n funcie de numrul de coloane i linii pe care le ntoarce subcererea) a subcererilor se aplic att pentru subcererile necorelate ct i pentru cele corelate.

Subcererile pot fi mprite n dou categorii, n funcie de modul n care sunt corelate cu cererea principal (cerea care cuprinde subcererea):

  • Subcereri necorelate: rezultatul subcererii nu este condiionat de valorile din cererea principal
  • Subcereri corelate: rezultatul subcererii este condiionat de valorile din cererea principal

Cnd folosim subcereri, trebuie s respectm cteva reguli:

  • Subcererea trebuie s fie inclus ntre paranteze;
  • Pentru cazurile n care subcererea se afl n clauza WHERE, sau HAVING, aceasta trebuie s fie n partea dreapt a condiiei;
  • Expresiile din lista de expresii ale subcererii trebuie s fie n aceeai ordine ca expresiile din lista de condiii din clauza WHERE sau HAVING a cererii principale (avnd acelai tip i numr de expresii);
  • Subcererile nu pot fi ordonate, deci nu conin clauza ORDER BY
  • Clauza ORDER BY apare la sfritul cererii principale.
  • Subcererile necorelate sunt executate de la cea mai adnc imbricare pn la nivelul principal de mbricare;
  • Subcererile pot folosi funcii de grup i clauza GROUP BY;
  • Subcererile pot fi nlnuite cu predicate multiple AND sau OR n aceeai cerere extern n clauze WHERE sau HAVING ale acesteia;
  • n subcereri se pot folosi operatori de mulimi;
  • Subcererile pot fi imbricate pn la nivelul 255;
  • n clauza SELECT o subcerere trebuie s ntoarc o singur valoare.

Subcereri necorelate n clauza WHERE

  • Subcererile necorelate sunt subcereri care nu au o legtur de asociere ntre expresiile cererii exterioare i cele ale cererii interioare.
  • O subcerere necorelat se execut o singur dat i se execut prima n momentul cnd ncepe execuia cererii principale.
  • Subcererea necorelat n clauza WHERE se construiete prima pentru a verifica condiia din caluza WHERE.
  • Condiia din subcererea necorelat din clauza WHERE se verific pe datele din tabela folosit n cererea principal.
Subcereri necorelate care ntorc o valoare n clauza WHERE
SELECT 
		[table_1.]expr_1, 
		[table_1.]expr_2,
		..., 
		[table_1.]expr_n
FROM table_1
WHERE [NOT] [table_1.]expr_k OPERATOR
	(SELECT [table_2.]expr_i 
	FROM table_2
	[WHERE conditions_1]
	[HAVING conditions_2])

[table_1.]expr_k sunt expresii pe coloanele(sau doar o coloan) care aparin tabelului table_1, k=1..n;
[table_2.]expr_i este o expresie pe coloanele(sau doar o coloan) care aparin tabelului table_2;
OPERATOR este mulimea operatorilor de comparaie {>, >=, =, <=, <};
conditions_1 sunt condiiile din clauza WHERE (opional) a subcererii;
conditions_2 sunt condiiile din clauza HAVING (opional) a subcererii;

Subcereri necorelate care ntorc o coloan n clauza WHERE

Sunt subcereri care ntorc mai multe valori i folosesc operatorii IN i NOT IN. n cazul n care se folosesc operatori de comparaie n locul operatorului IN (NOT IN), atunci se genereaz o eroare. Observatie:

Operatorul IN (NOT IN) se poate folosi n loc de operatori de comparaie, dar nu i invers. 

SELECT 
		[table_1.]expr_1, 
		[table_1.]expr_2,
		..., 
		[table_1.]expr_n
FROM table_1
WHERE [table_1.]expr_k  [NOT] IN
	(SELECT [table_2.]expr_i 
	FROM table_2
	[WHERE conditions_1]
	[[GROUP BY expressions] [HAVING conditions_2]])

Subcereri necorelate care ntorc o linie n clauza WHERE
  • Sunt cereri care ntorc mai multe colonale dar o singur linie;
  • Folosesc operatorii IN i NOT IN
  • Numrul i tipul expresiilor (coloanelor) din clauza WHERE a cererii principale trebuie s coincid cu cele din subcerere.
SELECT  [table_1.]expr_1, [table_1.]expr_2,..., [table_1.]expr_n
FROM table_1
WHERE ([table_1.]expr_i1, ..., [table_1.]expr_im)  [NOT] IN
	(SELECT  [table_2.]expr_j1, ..., [table_1.]expr_jm
	FROM table_2
	[WHERE conditions_1]
	[HAVING conditions_2])

Subcereri necorelate care ntorc mai multe linii n clauza WHERE
  • Aceste subcereri au sintaxa asemntoare cu subcererile care ntorc o singur linie;
  • Aceste subcereri ntorc mai multe linii i mai multe coloane, motiv pentru care se mai numesc i cereri care ntorc o tabel;
  • Numrul expresiilor(coloanelor) din clauza WHERE a cererii principale trebuie s coincid cu numrul de expresii din ntoarse de subcerere i trebuie s fie de acelai tip.
SELECT  [table_1.]expr_1, [table_1.]expr_2,..., [table_1.]expr_n
FROM table_1
WHERE ([table_1.]expr_i1, ..., [table_1.]expr_im)  [NOT] IN
	(SELECT  [table_2.]expr_j1, ..., [table_1.]expr_jm
	FROM table_2
	[WHERE conditions_1]
	[[GROUP BY expressions] [HAVING conditions_2]])

Subcereri corelate n clauze WHERE

Subcererile corelate se execut o singur dat pentru fiecare linie candidat prelucrat de cererea principal. O subcerere corelat se join-eaz cu cererea exterioar prin folosirea unei coloane a cererii exterioare n clauza predicatului cererii interioare. 

SELECT  [table_1.]expr_1, [table_1.]expr_2,..., [table_1.]expr_n
FROM table_1
WHERE ([table_1.]expr_i1, ..., [table_1.]expr_im)  [NOT] IN
	(SELECT  [table_2.]expr_j1, ..., [table_1.]expr_jm
	FROM table_2
	WHERE [table_2.]expr_x OPERATOR [table_1.]expr_y [{AND|OR} ...]
	[[GROUP BY expressions] 
	[HAVING [table_2.]expr_t OPERATOR [table_1.]expr_z [{AND|OR} ...]
]])

Pentru o subcerere corelat, subcererea se execut de mai multe ori, cte o dat pentru fiecare linie prelucrat de cererea principal, deci cererea interioar este condus de cererea exterioar; Paii de execuie a unei subcereri corelate sunt:

  • Se obine linia candidat prin procesarea cererii exterioare;
  • Se execut cererea interioar corelat cu valarea linie candidat;
  • Se folosete valoarea rezultatului din cererea interioar pentru a prelucra linia candidat;
  • Se repet pn nu mai rmne nicio linie candidat.

Dei o subcerere corelat se execut de mai multe ori, acest lucru nu implic c subcererile corelate sunt mai ineficiente dect subcererile necorelate.

Subcereri pe tabel temporar (n clauza FROM)

  • Aceste subcereri se ntlnesc n cazul n care se folosete o subcerere la nivelul clauzei FROM;
  • n clauza FROM se pot folosi doar subcereri necorelate;
  • Corelarea dintre tabele i tabelele temporare din clauza FROM se face folosind metode de join.
SELECT t1.expr_1, ..., t1.expr_n, t2.expr_1,..., t2.expr_m
FROM table_1 t1
	{,|[[INNER|CROSS|NATURAL]|[FULL|RIGHT|LEFT] [OUTER] ]] JOIN}
	(SELECT ti.expr_1, ... , ti.expr_s
	FROM table_2
	[WHERE nested_where_conditions]
	[[GROUP BY nested_expresions] 
		[HAVING nested_having_conditions]]
	) t2 
	[ON on_conditions]
	[WHERE where_conditions]
	[[GROUP BY expresions] 
		[HAVING having_conditions]]
[ORDER BY {COLUMNS|expressions|possition} [ASC|DESC]]
