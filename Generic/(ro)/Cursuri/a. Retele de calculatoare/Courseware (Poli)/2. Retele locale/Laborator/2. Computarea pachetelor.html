Mininet

Pentru a simula o reea virtual vom folosi Mininet. Mininet este un simulator de reele ce folosete n simulare implementri reale de kernel, switch i cod de aplicaii. Mininet poate fi folosit atat pe Linux cat si WSL2. 

sudo apt update
sudo apt install mininet openvswitch-testcontroller tshark python3-click python3-scapy xterm python3-pip
sudo pip3 install mininet

Dup ce am instalat Mininet, vom folosi urmtoarea comand pentru a crete dimensiunea fontului n terminalele pe care le vom deschide. 

echo "xterm*font: *-fixed-*-*-*-18-*" >> ~/.Xresources
xrdb -merge ~/.Xresources

Cnd o s rulm simularea, e posibil s ntlnii urmtoarea eroare: Exception: Please shut down the controller which is running on port 6653:. Pentru a rezolva problema, va trebui s rulai pkill ovs-test.

01. [15p] Implementare Hub

Vom implementa in Python funcionalitatea de baza a unui Hub. Cum am vazut si in cadrul laboratorului anterior, pentru fiecare frame primit, hub-ul l trimite pe toate interfeele (link sau port). Mai jos gasim o descriere in pseudocod: 

while hub is powered on:
    for each port:
        if there is incoming data:
            read the incoming data
            for each other port:
                send the data to that port

In acest exerciiu vom implementa un Hub in Python pornind de la un API simplu de primire si trimitere de frame-uri Ethernet. Vom porni de la scheletul de la urmatoarea adresa. Tot acolo, gasiti in README.md detalii despre rularea scheletului. Implementarea se va face in hub.py, in dreptul celor doua TODO-uri. 

# data este de tip byte array
interface, data, length = recv_from_any_link()
send_to_link(interface, data, length)

02. [15p] Introducere n Packet Tracer

Descrcai arhiva de instalare Packet Tracer 8.2 de aici.

Tutoriale (old):

  • Packet Tracer 6.2 Windows
  • Packet Tracer 6.2 Linux

Packet Tracer este un utilitar cu ajutorul cruia se pot simula topologii de reea folosind hub-uri, rutere, switch-uri i calculatoare. Vom prezenta principalele componente din spaiul de lucru:

  1. La apsare vor aprea mai multe modele de rutere n dreapta acestuia. Este suficient s alegei modelul Router-PT i s l plasai n zona de lucru (partea alb din mijloc) oriunde dorii.
  2. La apsare vor aprea mai multe modele calculatoare n dreapta acestuia. Este suficient s alegei modelul PC-PT i s l plasai n zona de lucru (partea alb din mijloc) oriunde dorii.
  3. Se alege tipul legturii (UTP straight, UTP cross-over, conexiune serial, etc.). Este suficient s alegei primul tip de cablu (cel marcat Automatically Choose Connection Type, cu iconia unui fulger portocaliu), aceasta punnd automat tipul legturii dintre dou echipamente. Dup selecia tipului de cablu apsai pe echipamentele care se doresc a fi unite. n unele cazuri vei avea de ales portul echipamentului (ex. FastEthernet 0/1, FastEthernet 0/2, etc.)
  4. La apsare vor aprea mai multe modele de switch-uri n dreapta acestuia. Este suficient s alegei modelul Switch-PT i s l plasai n zona de lucru (partea alb din mijloc) oriunde dorii.
  5. La apsare vor aprea mai multe modele de hub-uri n dreapta acestuia. Este suficient s alegei modelul Hub-PT i s l plasai n zona de lucru (partea alb din mijloc) oriunde dorii.
  6. Putei folosi butonul Delete pentru a terge istoria pachetelor trimise.
  7. n chenarul din dreapta jos va aprea pachetul trimis i status-ul recepiei (Last Status).
  8. Se poate selecta modul de operare: Realtime (modul implicit) sau Simulation (se apas pe tab-ul din spatele Realtime pentru a-l activa). n modul Simulation se poate vedea pas cu pas cum circul pachetele ntre echipamentele active. n modul Simulation trimitei un pachet i apoi apsai pe butonul Auto Capture/Play.
  9. Se poate trimite un pachet ntre dou entiti de nivel 3 (calculator, router). Dup selectare, apsai pe echipamentele ntre care vrei s trimitei pachete.

Testare conectivitate

  • Creai o topologie cu 2 switch-uri legate ntre ele i cte 2 calculatoare legate la fiecare din switch-uri.
  • nainte de a putea trimite pachete, trebuie configurate adresele IP pe fiecare din calculatoare. La apsarea unui click pe calculator, va aprea o fereastr prin care putei configura calculatoarele. Mergei pe tab-ul Desktop i apsai pe iconia IP Configuration.
  • Completai cmpul IP Address cu o adres IP din intervalul 192.168.1.1 - 192.168.1.5 i cmpul Subnet Mask cu valoarea 255.255.255.0. Restul cmpurilor rmn goale.
  • Verificai conectivitatea ntre fiecare 2 calculatoare folosind facilitatea 9 prezentat mai sus.

Dac ai introdus o comand inexistent n CLI este foarte posibil s se blocheze pentru o perioad scurt de timp terminalul i s v afieze un mesaj asemntor celui de mai jos: 

R1# end
Translating "end"...domain server (255.255.255.255)

n acest caz folosii combinaia de taste: CTRL + SHIFT + 6 sau CTRL + ALT + 6

03. [15p] Switch vs. Hub

Topologie: download.

  • Vrem s urmrim diferenele ntre switch i hub. Vom simula transmiterea de pachete ntre staii folosind o topologie PacketTracer n care se afl 2 reele cu cte 4 calculatoare fiecare. Diferena ntre topologii este dispozitivul de interconectare: una este cu un switch, iar cealalt este cu un hub.
  • Ateptai ca porturile din switch s lumineze cu verde (nu cu portocaliu) i apoi trimitei pachete ntre oricare 2 calculatoare n cele 2 topologii pentru a v asigura c exist conectivitate. (lumineaz cu portocaliu timp de circa 30 de secunde pn cnd se definitiveaz rularea STP - Spanning Tree Protocol)
  • Dup ce ati verificat existena conectivitii, intrai n modul Simulation i trimitei cte un pachet ntre 2 calculatoare. Facei acest lucru pentru ambele reele. Observai ce pachete sunt generate n reea n fiecare caz.
  • Ce putei spune despre diferena dintre switch i hub?
  • La ce nivel din stiva OSI ruleaz fiecare?

04. [15p] Popularea tabelei de comutare

Topologie: download

  • Dorim s urmrim modul n care se populeaz tabela de comutare a unui switch. Topologia este format dintr-un switch i 4 staii. ncrcai fiierul de topologie n Packet Tracer i ateptai ca porturile switch-ului s lumineze verde n loc de portocaliu (s fi rulat STP).
  • Dup cum ai observat de la exerciiul anterior, switch-ul comut pachetele doar ctre destinaie, fcnd acest lucru pe baza unei tabele de comutare (MAC Address Table).
  • Pentru configurarea unui switch, apsm pe iconia sa din fereastr alb din Packet Tracer. Va aprea o fereastr prin care putem configura switch-ul. Singurul tab de interes este cel denumit CLI (Command Line Interface). Apsai tasta Enter cel puin o dat ca s vedei unde este cursorul. Acest CLI este specific Cisco si este format dup cum urmeaz:
    • literele pn la caracterul > (mai mare) reprezint hostname-ul echipamentului
    • caracterul > semnific faptul c suntei autentificat ca un utilizator neprivilegiat (fr drepturi administrative)
  • Obtineti drepturi administative pentru a afisa tabela de comutare/MAC a switch-ului si observati ca este goala.

Tabela de comutare

  • Tabela de comutare conine mai multe coloane dintre care dou sunt de interes:
    • coloana Mac Address specific adresa MAC
    • coloana Ports specific portului switch-ului
  • Tabela este, aadar, o asociere ntre o adres MAC i un port. Spunem c tabelele au, pentru fiecare intrare:
    • o component de potrivire i de cutare (match) - se caut o adres MAC
    • o component de aciune - pe baza rezultatului obtinut in urma operatiei de match se trimite pachetul pe portul aferent.
  • Partea de cutare poate aprea o singur dat, partea de aciune de mai multe ori:
    • putem avea mai multe adrese MAC aferente unui port
    • o adres MAC poate aprea cel mult o dat n tabela de comutare.
  • Aflati numele/numrul fiecrui port al unui switch si adresa MAC a unei statii.
  • Trimitei un pachet de la PC0 la PC1. Afiai din nou tabela de comutare.
  • Observai adugarea a dou intrri:
    • O intrare este generat la trimiterea pachetului PC0 la PC1 - conine adresa MAC a staiei PC0 i portul pe care aceasta este legat la switch
    • A doua este generat la ntoarcerea pachetului de la PC1 la PC0 - conine adresa MAC a staiei PC1 i portul pe care aceasta este, la rndul ei, legat la switch.
  • Trimitei un pachet de la PC1 la PC2. Afiai din nou tabela de comutare. De ce apare doar o singur intrare, nu dou?
  • Trimitei un pachet de la PC2 la PC3. Afiai din nou tabela de comutare. Ce observai?
  • Curai tabela de comutare.
  • Intrai n modul Simulation i trimitei un pachet de la PC1 la PC3. Ce observai? Urmrii n fereastra Event list un sumar al pachetelor trimise.
  • Mai trimitei nc un pachet de la PC1 la PC3. De ce s-a comportat diferit de data aceasta? Afiai din nou tabela de comutare.
  • Curai tabela de comutare.
  • Intrai n modul Simulation i trimitei un pachet de la PC1 la PC3. La fiecare pas al simulrii vizualizai tabela de comutare a switch-ului. Discutai cu asistentul dac avei ntrebri legate de popularea tabelei de comutare.

05. [20p] Coninutul tabelei de comutare

Topologie: download

Analiza tabela de comutare

  • Ne propunem s urmrim coninutul tabelei de comutare ntr-un scenariu cu 2 switch-uri.
  • La fiecare switch sunt conectate cte 2 calculatoare. ncrcai fiierul topologie n PacketTracer i ateptai ca switch-urile s ruleze STP. dup rularea STP, porturile switch-urilor vor avea culoarea verde.
  • Trimitei pachete ntre orice 2 calculatoare din reea i vizualizai tabela de comutare pe ambele switch-uri.
  • Observai c un port poate avea asociate mai multe adrese MAC. Portul unui switch care are legtura cu cellalt switch cuprinde trei adrese MAC:
    • adresa MAC a portului celuilalt switch
    • adresele MAC ale staiilor conectate la cellalt switch

Analiza tabela de comutare dupa decuplarea unui echipament

  • Afiai tabela de comutare a switch-ului Switch0 i reinei portul asociat adresei MAC a staiei PC1.
  • Decuplai staia PC1 de la reea apsnd pe beculeul verde de pe legtura la Switch0 a acestuia urmat de tasta Escape (un alt mod de a terge o legtur se poate obine prin folosirea butonului n form de X din dreapta, urmat de apsarea pe elementul care se dorete a fi ters).
  • Dup decuplarea calculatorului, urmrii tabela de comutare i observai pierderea intrrii aferente.

Analiza tabela de comutare dupa reconectarea unui echipament

  • Folosii un cablu Copper Straight-Through pentru a conecta staia PC1 la cellalt port disponibil al switch-ului Switch0 (diferit de cel iniial).
  • Dup rularea STP pe switch, trimitei un pachet ntre PC1 i PC0.
  • Observai popularea tabelei de comutare cu o nou intrare corespunztoare adresei MAC a staiei PC1. O adres MAC nu poate aprea de dou ori n tabela de comutare a unui switch.

06. [20p] Intrri statice n tabela de comutare

Topologie: download

  • Pn n acest punct am observat cum se poate popula dinamic o tabela de comutare a unui switch. Ai vzut c n tabela de comutare intrrile erau marcate cu DYNAMIC n dreptul coloanei Type.
  • Tabela de comutare se poate popula i static cu intrri predefinite pentru un anumit port. Dorim s configurm intrri statice n tabela de comutare a unui switch.
  • Pentru a putea configura un echipament Cisco, trebuie s intrm n modul de configurare.
  • Acest mod l vei recunoate prin adugarea sufixului (config) la prompt-ul CLI-ului.
  • n acest mod putei doar s realizai configurri, nu putei vizualiza parametrii switch-ului (ex. afiarea tabelei de comutare).

Configurare static n tabela de comutare

  • Topologia conine un switch i 4 calculatoare. ncrcai fiierul coninnd topologia n PacketTracer i ateptai rularea STP.
  • Vom configura o intrare static pentru staia PC1, staie conectat pe portul FastEthernet1/1 (sau Fa1/1) din switch i care are adresa MAC 0001.639D.68E3.
  • Trimitei un pachet de la staia PC0 la staia PC1. Afiai tabela de comutare.
  • Se observ tipul STATIC pentru intrarea cu adresa MAC a staiei PC1 i intrarea de tipul DYNAMIC pentru adresa MAC a staiei PC0.
  • Creai nc o intrare static pentru staia PC2 dup modelul de mai sus. Verificai conectivitatea cu toate celelalte staii.
  • Golii tabela de comutare.
  • Creai o intrare static pentru portul din switch aferent staiei PC3, dar n loc sa punei adresa MAC a staiei PC3, punei o adres MAC care nu exist n reeaua local.
  • Afiai tabela de comutare. Testai conectivitatea staiei PC3 cu celelalte staii i afiai din nou tabela de comuntare.
  • Observai prezena a dou intrri MAC pe acelai port, una static (introdus mai devreme) i alta dinamic (adresa MAC a staiei PC3).
  • O intrare static are avantajul permanenei. n absena traficului, o intrare dinamic este tears din tabela de comuntare dup cteva minute. O intrare static poate fi tears doar explicit, prin folosirea comenzii clear mac-address-table.
  • Dac dorim s configurm acces exclusiv al staiei PC3 n tabela de comutare, trebuie folosite comenzi de port security.
