Exerciii

00. [M][5p] Lucru cu bii

  • Reprezentarea adreselor IP se face n grupuri de cte 8 bii. Drept pentru care, v sunt necesare cunotine bune de lucru n binar.
  • Atunci cnd facei reprezentarea n binar a unui numr cel mai bine este s l descompunei n puteri ale lui 2. De exemplu:
    • 17 = 16 + 1 = 24 + 20; rezult reprezentarea (n binar pe 8 bii) 00010001
    • 173 = 128 + 32 + 8 + 4 + 1 = 27 + 25 + 23 + 22 + 20; rezult reprezentarea (n binar pe 8 bii) 10101101
  • Sunt utile abilitile de a face rapid reprezentarea rapid a numerelor care se gsesc n jurul puterilor lui 2. De exemplu:
    • pentru 64 = 01000000
      • 63 este 64-1, un numr cu muli bii de 1 (6 bii): 00111111
      • 62 este 63-1, un numr cu muli bii de 1 mai puin ultimul (5 bii de 1 i un bit de 0): 00111110
      • 65 este 64+1, se adaug ultimul bit de 1 la reprezentarea lui 64: 01000001

01. [M][5p] Adres IP i masc de reea

  • Ne propunem s sistematizm noiunile de adres IP, masc de reea, adres de reea i adres de broadcast.
  • Pentru asigurarea conectivitii avem nevoie de o adres de nivel 3, numit i adres IP. Pe fiecare echipament cu suport de nivel 3 (staii, rutere, switch-uri layer 3) avem nevoie de adrese IP.
  • n cazul unei adrese IP, vom configura, tot timpul, urmtoarele:
    • adresa IP - 4 grupuri a cte 8 bii1). Exemplu: 192.168.100.2002)
    • masca de reea (subnet mask) - 4 grupuri a cte 8 bii, cu proprietatea c se ncepe cu bitul 1, iar toi biii de 1 sunt consecutivi, alternana 0/1 fiind interzis. De exemplu 11111111.00000000.00000000.00000000 este o masc de reea valid, iar 11000001.00000000.00000000.0000000 este o masc nevalid. Pentru a uura citirea mtii acestea se scrie n zecimal, similar adresei IP: 11111111.00000000.00000000.00000000 = 255.0.0.0. Datorit proprietii speciale n care biii de 1 sunt consecutivi o alt form n care vei mai gsi specificat masca de reea este forma prefixat: /X, unde X reprezint numrul de bii de 1: 11111111.00000000.00000000.00000000 = 255.0.0.0 = /8.
  • Pornind de la adresa IP i masca de reea putem identifica dou alte proprieti ale unei reele (pentru exemplificare vom folosi adresa IP 192.168.100.200/255.255.255.0):
    • adresa de reea - se obine fcnd I-logic ntre biii adresei IP i biii mtii de reea
      • 192.168.100.200 & 255.255.255.0 = 192.168.100.0
    • adresa de broadcast - se obine fcnd SAU-logic ntre biii adresei IP i biii din complementul mtii de reea (complementul se obine inversnd valoarea biilor de pe fiecare poziie)
      • 192.168.100.200 | 0.0.0.255 = 192.168.100.255
  • Atunci cnd cunoatem adresa IP i masca de reea i vrem s obinem adresa de reea i adresa de broadcast, este util s folosim masca de reea pentru a mpri adresa IP n dou:
    • O parte de subreea, care se ntinde pe ci bii de 1 are masca de reea. E vorba de 24 de bii, pentru o masc /24 (sau 255.255.255.0) sau 16 bii pentru o masc /16 (sau 255.255.0.0) sau 20 de bii pentru o masc /20 (sau 255.255.240.0).
    • O parte de staie (sau parte de host) care se ntinde pe restul spaiului (32 - numrul de bii de 1 ai mtii de reea). E vorba de 8 bii pentru o masc /24 (32-24 = 8) sau de 16 bii pentru o masc /16 (32-16 = 16) sau de 12 bii pentru o masc /20 (32-20 = 12).
  • Pe aceast mprire vom obine aceeai pentru adresa 192.168.100.200/24 aceleai valori precum cele calculate mai sus, lucru reflectat i n figura de mai jos.

  • S obinem adresa de reea i adresa de broadcast pentru adresa 172.16.200.100/20.
  • Transformm adresa ntr-o adres hibrid punnd bii de 0 acolo unde se gsete masca de reea: al treilea octet din cei patru ai adresei IP: 172.16.1100|1000.xxxxxxxx.
  • Am folosit operatorul | (pipe) pentru a separa partea de reea (primii 20 de bii, afereni reelei) de partea de staie (host) (ceilali bii (32-20 = 12 bii) afereni staiei). Nu sunt relevani pentru calculul nostru biii ultimului octet aa c am pus xxxxxxxx n locul lor.
  • Adresa de reea are toi biii de staie pui pe 0, deci va fi 172.16.1100|0000.00000000. Rezult adresa de reea 172.16.192.0/20.
  • Adresa de broadcast are toi biii de staie pui pe 1, deci va fi 172.16.1100|1111.11111111. Rezult adresa de broadcast 172.16.207.255/20.

03. [M][5p] Utilitate masc de reea

Topologie: download

  • Topologia conine 3 calculatoare legate printr-un switch i au configurate adresele IP. Dup ce toate legturile devin active (toate beculeele devin verzi), trimitei cte un pachet ntre oricare dou staii.
  • Observai c nu exist conectivitate ntre staia PC1 i celelalte dou staii (PC0 si PC2).
  • Vizualizai adresele IP i mtile de reea configurate pe fiecare din staii.
  • Motivul pentru care nu funcioneaz transmiterea pachetelor de la PC1 este pentru c staia PC1 se afl n alt reea fa de staiile PC0 i PC2.
  • Dei aparent este n aceeai reea, difer masca de reea. Vom demonstra acest lucru calculnd adresele de reea:
    • PC0: 172.16.10.10 & 255.255.0.0 (/16) = 172.16.0.0
    • PC1: 172.16.20.20 & 255.255.255.0 (/24) = 172.16.20.0
    • PC2: 172.16.30.30 & 255.255.0.0 (/16) = 172.16.0.0
  • Se observ c staiile PC0 i PC2 fac parte din aceeai reea, pe cnd PC1 se afl n alt reea, lucru indicat de masca de reea.

Apartenena unei adrese IP la o reea se poate decide doar folosind att adresa IP ct i masca de reea. O configuraie fr masc de reea este incomplet.

04. [M][5p] Numr de staii n reea

  • Masca de reea face separaia ntre partea de reea i partea de staie. Numrul de bii pentru partea de staie reflect adresele IP posibile pentru staii.
  • Dou dintre aceste adrese, ns, nu sunt asignabile staiilor (nu pot fi configurate pe staii). Este vorba de prima adres IP a spaiului i ultima adres IP a spaiului posibil.
  • Prima adres IP a spaiului de adrese conine doar bii de 0 pentru partea de host i este adresa de reea.
  • Ultima adres IP a spaiului de adrese conine doar bii de 1 pentru partea de host i este adresa de broadcast.
  • Dac avem masca de reea /24, adic 8 bii (32-24) pentru partea de staie, vom avea un spaiul posibil de 28 = 256 adrese. Din aceste adrese 2 nu sunt folosibile (adresa de reea i adresa de broadcast) i vom avea, aadar 254 de adrese asignabile.
  • n general, dac avem N bii alocai pentru partea de staie, vom avea 2N-2 adrese asignabile.
  • Rspundei la ntrebrile de mai jos:
    1. Cte adrese asignabile se gsesc n reeaua 10.10.0.0/16?
    2. Cte adrese asignabile se gsesc n reeaua 15.16.192.0/20?
    3. Cte adrese asignabile se gsesc n reeaua 1.2.3.4/30?
    4. Care este masca de reea a celei mai mici reele care s cuprind 25 de adrese asignabile?
    5. Care este masca de reea a celei mai mici reele care s cuprind 62 de adrese asignabile?
    6. Care este masca de reea a celei mai mici reele care s cuprind 127 de adrese asignabile?

05. [M][5p] Prezentare subnetare

  • n reele mari, se pornete de la un spaiu de adres oferit de o autoritate i apoi se fac subreele la acest spaiu. Subreelele vor acoperi zone/departamente diferite. Procesul de mprire n subreele se cheam subnetting sau subnetare.
  • n forma simpl, se mpart n subreele egale, de dimensiune fix.

Tutorial

  • Vom realiza subnetarea reelei 78.78.32.0/22 n 4 subreele
  • n forma binar hibrid, adresa reelei mari este 78.78.001000|00.00000000/22. Am folosit caracterul | (pipe) pentru a separa partea de reea de partea de host.
  • Avem 4 subreele deci avem nevoie de 2 bii suplimentari pentru indicarea reelei: 00, 01, 10, 11. nseamn c subreelele vor avea masca /22 + 2 bii =/24. Adic vor avea 28-2 = 254 staii fiecare.
  • Cele 4 subreele vor avea, respectiv adresele:
    • 78.78.00100000.00000000/24 = 78.78.32.0/24
    • 78.78.00100001.00000000/24 = 78.78.33.0/24
    • 78.78.00100010.00000000/24 = 78.78.34.0/24
    • 78.78.00100011.00000000/24 = 78.78.35.0/24
  • Adresa de broadcast a fiecrei subreele va fi, respectiv:
    • 78.78.00100000.11111111/24 = 78.78.32.255/24
    • 78.78.00100001.11111111/24 = 78.78.33.255/24
    • 78.78.00100010.11111111/24 = 78.78.34.255/24
    • 78.78.00100011.11111111/24 = 78.78.35.255/24
  • Prima adres asignabil din cadrul fiecrei subreele va fi, respectiv:
    • 78.78.00100000.00000001/24 = 78.78.32.1/24
    • 78.78.00100001.00000001/24 = 78.78.33.1/24
    • 78.78.00100010.00000001/24 = 78.78.34.1/24
    • 78.78.00100011.00000001/24 = 78.78.35.1/24
  • Ultima adres asignabil din cadrul fiecrei subreele va fi, respectiv:
    • 78.78.00100000.11111110/24 = 78.78.32.254/24
    • 78.78.00100001.11111110/24 = 78.78.33.254/24
    • 78.78.00100010.11111110/24 = 78.78.34.254/24
    • 78.78.00100011.11111110/24 = 78.78.35.254/24

09. [M][5p] Prezentare VLSM

  • S presupunem c un institut primete un spaiu 17.18.19.0/24. Decizia organizatoric dicteaz ca acel spaiu s fie mprit n trei subreele corespunztoare pentru trei departamente:
  1. A: o subreea cu un numr de 100 de staii
  2. B: o subreea cu 60 de staii
  3. C: o subreea cu 50 de staii
  • Cnd facem subnetting folosind VLSM, sortm n ordine descresctoare subreelele dup numrul de staii. Pornim de la subreeaua cea mai mare i apoi continum, pas cu pas, la reelele mai mici.
  • Prima subreea are 100 de staii, deci are nevoie de 7 bii pentru partea de host, adic de o masc /25. mprim reeaua mare n dou subreele:
    1. 17.18.19.0/25
    2. 17.18.19.128/25
  • Prima subreea o alocm primul departament: A - 17.18.19.0/25.
  • A doua subreea (17.18.19.128/25) o mprim n continuare.
  • Trebuie s folosim spaiul rmas pentru celelalte dou subreele (B i C) cu cte 60, respectiv 50 de staii.
  • Avem nevoie de 6 bii pentru partea de host a fiecrei subreele, adic de o masc /26. mprim noua subreea n dou subreele:
    1. 17.18.19.128/26
    2. 17.18.19.192/26
  • Alocm fiecare dintre cele dou subreele rezultate celor dou departamente: B - 17.18.19.128/26 i C - 17.18.19.192/26.
  • ntruct spaiul iniial a fost mprit n subreele cu masc diferit, numim acest process Variable Length Subnet Mask (VLSM).

12. [A][10p] Configurare adrese IP i ruter n Packet Tracer

Topologie: download

  • Topologia este format dintr-un ruter i 2 staii conectate la 2 interfee diferite ale ruterului prin switch-uri.
  • Pentru a avea conectivitate ntre staii i ruter trebuie sa configurm adrese IP pe fiecare din ele.
  • Vom folosi 2 clase, cte una pentru fiecare reea reprezentat de cele 2 interfee ale ruterului:
    • 192.168.0.0/24
    • 10.0.0.0/24
  • Configurai pe staia PC0 adresa IP 192.168.0.10 cu masca 255.255.255.0, iar pe PC1 adresa IP 10.0.0.10 cu masca 255.255.255.0.
  • Pe interfaa Fa0/0 a ruterului vom configura adresa IP 192.168.0.1 cu masca 255.255.255.0:
Router>enable
Router#configure terminal
Router(config)#interface fa0/0
Router(config-if)#no shutdown 
Router(config-if)#ip address 192.168.0.1 255.255.255.0
Router(config-if)#exit
Router(config)#exit
Router# ping 192.168.0.10
  • Configurai interfaa Fa1/0 a ruterului asemntor interfeei Fa0/0 folosind adresa IP 10.0.0.1 cu masca de reea 255.255.255.0.

Pentru a ridica interfaa, folosii comanda no shutdown.

De pe ruter, verificai conectivitatea la staia PC1 folosind comanda ping.

Putei realiza verificarea folosind i mesajele din Packet Tracer.

13. [A][BONUS - 10p] Configurare VLSM n Packet Tracer

Topologie: download

  • Topologia conine un ruter care interconecteaz 3 reele, fiecare reea avnd cte dou staii conectate la un switch.
  • Folosind clasele obinute la exerciiul 9, configurai cele 3 reele astfel:
    • reteaua cu statiile PC0, PC1 si interfata aferenta de pe ruter sa corespunda primei clase de adrese
    • reteaua cu statiile PC2, PC3 si interfata aferenta de pe ruter sa corespunda celei de-a doua clase de adrese
    • reteaua cu statiile PC4, PC5 si interfata aferenta de pe ruter sa corespunda celei de-a treia clase de adrese
  • Configurarea se va realiza dup urmtoarea regul:
    • prima adres alocabil din clas va fi configurat pe interfaa ruterului
    • alte dou adrese la alegere din clas vor fi configurate pe cele 2 staii
  • Verificai conectivitate ntre staiile din aceeai reea i ruter folosind comanda ping de pe ruter sau pachetele puse la dispozitie de Packet Tracer.

Interfeele ruterului trebuie s fie pornite; folosii comanda no shutdown. Beculeele trebuie s fie verzi.

Pentru a afla pe ce interfa a ruterului se conecteaz o legtur fizic (Fa1/0, Fa4/0 etc.) plasai mouse-ul (hover) peste beculeul aferent legturii i interfeei.
